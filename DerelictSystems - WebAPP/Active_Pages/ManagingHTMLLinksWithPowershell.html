    Managing HTML Links With Powershell

    So first things first. Let's layout the file format so we can understand the code a bit easier later on. First, only HTML and CSS is used. This means we have a rigid structure that must be adheared to with absolute links to pages and resources. Here is a high level view.



    We can see the root folder system here has several key folders. All of which will need to be maintained as changes are made.

    • Active pages will contain the two blogs most recently written.
    • Archive will contain pages bumped off when new blogs are written.
    • Assets contains images like the one above.
    • CSS contains code that governs how the page looks. More than that, it can "create" the appearance of action as well!
    • Old Resources will contain exactly that. Discarded or test objects
    • Index is our Home Page!
    • The LinkManagement.csv file is our "Database". This will contain our file and link information and object relations.

    Now, you may ask why I'm putting myself through this rather than learning some basic modern coding techniques. The short answer is "Because I can!". I want to showcase the fact that anything can be automated if you throw enough time at it! Joking aside. This is a technical challenge that I find much more interesting then banging out "How to I web 101" into the latest chatbot. Okay, with the hierarchy tackled, lets dive into the issues we face and how this powershell web management framework addresses thos issues.

    • How do we track Objects and their relationships?
    • What if you need/want to change a file name?
    • How do you migrate pages from one folder to another without breaking the links?
    • What else are you planning to use this management framework for?
    • I just want to "borrow" your code. Where can I find it?

    First up, How do we track objects and their relationships? The same way I've started this section of course! With a numerical value system. Just like any other relational database system, we have to track items using unique values.

    These Object ID's allow us to uniquely identify each object and we can use these values in relation to each other.
